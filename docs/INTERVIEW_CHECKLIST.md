## 🧠 Technical Interview Checklist

### 🟢 Easy Problems Checklist (First 10-15 min)

**Arrays & Strings**
- [ ] Can identify Two Pointers pattern
- [ ] Can identify Sliding Window pattern
- [ ] Can use HashMap for O(1) lookup
- [ ] Can handle edge cases (null, empty, single element)
- [ ] Can convert between String <-> char[]
- [ ] Can use StringBuilder for concatenation

**Linked Lists**
- [ ] Can handle null head/tail
- [ ] Can use dummy node technique
- [ ] Can detect cycles (Floyd's algorithm)
- [ ] Can reverse list (iterative & recursive)
- [ ] Can find middle (fast & slow pointers)

**Binary Search**
- [ ] Can identify sorted/monotonic property
- [ ] Can handle duplicates
- [ ] Can find first/last occurrence
- [ ] Can search in rotated array
- [ ] Can search on answer space

**Recursion**
- [ ] Can identify base cases
- [ ] Can define recursive relation
- [ ] Can trace call stack
- [ ] Can convert to iterative (if needed)
- [ ] Can add memoization

---

### 🟡 Medium Problems Checklist (20-30 min)

**Trees**
- [ ] Can do inorder/preorder/postorder traversal
- [ ] Can do level-order (BFS with queue)
- [ ] Can validate BST property
- [ ] Can find LCA (Lowest Common Ancestor)
- [ ] Can serialize/deserialize
- [ ] Can handle balanced tree check
- [ ] Can handle symmetric tree check

**Graphs**
- [ ] Can implement DFS (recursive/iterative)
- [ ] Can implement BFS (queue)
- [ ] Can detect cycles
- [ ] Can find connected components
- [ ] Can do topological sort (Kahn's/BFS)
- [ ] Can handle grid problems (direction arrays)
- [ ] Can handle visited set to avoid cycles

**Dynamic Programming**
- [ ] Can define state clearly
- [ ] Can write transition formula
- [ ] Can initialize base cases
- [ ] Can choose memoization vs tabulation
- [ ] Can optimize space (if possible)
- [ ] Can trace DP table for answer
- [ ] Can handle 1D & 2D DP problems

**Backtracking**
- [ ] Can define choice space
- [ ] Can define constraints
- [ ] Can define goal state
- [ ] Can implement choose-explore-unchoose
- [ ] Can prune search space
- [ ] Can handle duplicate elements
- [ ] Can generate all valid solutions

**Greedy**
- [ ] Can prove greedy choice property
- [ ] Can sort by key criteria
- [ ] Can handle interval problems
- [ ] Can use min/max heap
- [ ] Can prove local optimum → global optimum

**Heaps**
- [ ] Can use PriorityQueue (min/max heap)
- [ ] Can implement custom comparator
- [ ] Can find Kth largest/smallest
- [ ] Can merge K sorted lists
- [ ] Can maintain running median

---

### 🔴 Hard Problems (30-45 min)

**Advanced DP**
- [ ] Can optimize from O(n²) to O(n) or O(n log n)
- [ ] Can handle state compression
- [ ] Can handle bitmask DP
- [ ] Can handle digit DP
- [ ] Can handle probability DP

**Advanced Graphs**
- [ ] Can implement Dijkstra's algorithm
- [ ] Can implement Bellman-Ford
- [ ] Can find MST (Kruskal's/Prim's)
- [ ] Can handle strongly connected components
- [ ] Can handle bipartite matching

**System Design**
- [ ] Can estimate QPS / load
- [ ] Can design database schema
- [ ] Can choose appropriate data structures
- [ ] Can handle caching strategy
- [ ] Can handle scaling (horizontal/vertical)
- [ ] Can handle consistency trade-offs

## 💻 Coding Phase Checklist

### Before Coding (2-3 min)
- [ ] Clarify problem statement
- [ ] Ask about input/output format
- [ ] Ask about constraints (size, range)
- [ ] Ask about edge cases
- [ ] Think aloud about possible approaches
- [ ] Choose optimal approach (explain why)
- [ ] Outline algorithm steps verbally
- [ ] Estimate time/space complexity

### While Coding (15-25 min)
- [ ] Write clean, readable code
- [ ] Use meaningful variable names
- [ ] Add comments for complex logic
- [ ] Handle edge cases explicitly
- [ ] Write modular code (helper functions)
- [ ] Follow Java conventions
- [ ] Use proper indentation
- [ ] Don't write overly clever code
- [ ] Explain as you code (optional but helpful)

### After Coding (5-10 min)
- [ ] Dry run with sample input
- [ ] Test edge cases (null, empty, single, large)
- [ ] Test boundary conditions
- [ ] Verify time/space complexity
- [ ] Check for bugs (off-by-one, null pointer)
- [ ] Optimize if time permits
- [ ] Explain optimizations possible
